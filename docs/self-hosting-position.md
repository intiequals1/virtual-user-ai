# Self-Hosting Position — Virtual User AI

## Decision
Virtual User AI should be designed so that any user or organization can host it on:
- a local computer
- a private server
- a cloud VM or cloud instance
- a company-controlled infrastructure environment

## Product implication
This shifts the product from a pure SaaS-first model to a **self-hosting-first or hybrid deployment model**.

## Recommended positioning
The best commercial position becomes:
- self-hostable core
- optional managed cloud offering
- paid enterprise controls, support, governance, and deployment packages

## Why this is strong
- lower trust barrier
- better fit for privacy-sensitive users
- stronger fit for regulated environments
- easier adoption by technical users and teams
- avoids forcing all customers into one hosting model

## Revenue implication
Revenue should come less from mandatory hosting and more from:
- support
- enterprise features
- managed deployment
- domain packs
- governance and compliance tooling

## Technical implication
Codex and future implementation should preserve deployment flexibility:
- environment-based configuration
- no hard dependency on one cloud provider
- container-friendly structure
- local and server deployment paths
- optional external services instead of mandatory vendor lock-in
